# WCAG 2.2 Quick Reference

Quick reference for Web Content Accessibility Guidelines 2.2 Level AA success criteria.

## WCAG Principles (POUR)

1. **Perceivable** - Information and UI components must be presentable to users
2. **Operable** - UI components and navigation must be operable
3. **Understandable** - Information and UI operation must be understandable
4. **Robust** - Content must be robust enough for assistive technologies

## Level A (25 criteria) + Level AA (23 criteria) = 48 Total

### 1. Perceivable

**1.1 Text Alternatives**
- 1.1.1 Non-text Content (A) - All non-text content has text alternative

**1.2 Time-based Media**
- 1.2.1-1.2.5 (A/AA) - Captions, audio descriptions, transcripts

**1.3 Adaptable**
- 1.3.1 Info and Relationships (A) - Structure programmatically determined
- 1.3.2 Meaningful Sequence (A) - Reading order makes sense
- 1.3.3 Sensory Characteristics (A) - Not solely shape/size/color
- 1.3.4 Orientation (AA) - Works in any orientation
- 1.3.5 Identify Input Purpose (AA) - Autocomplete attributes

**1.4 Distinguishable**
- 1.4.1 Use of Color (A) - Color not sole indicator
- 1.4.3 Contrast (Minimum) (AA) - 4.5:1 text, 3:1 large text
- 1.4.11 Non-text Contrast (AA) - 3:1 UI components
- 1.4.12 Text Spacing (AA) - Text reflows at increased spacing
- 1.4.13 Content on Hover/Focus (AA) - Dismissible, hoverable, persistent

### 2. Operable

**2.1 Keyboard Accessible**
- 2.1.1 Keyboard (A) - All functionality via keyboard
- 2.1.2 No Keyboard Trap (A) - Can move focus away
- 2.1.4 Character Key Shortcuts (A) - Can be disabled/remapped

**2.2 Enough Time**
- 2.2.1 Timing Adjustable (A) - Can extend time limits
- 2.2.2 Pause, Stop, Hide (A) - Can pause animations

**2.4 Navigable**
- 2.4.1 Bypass Blocks (A) - Skip navigation mechanism
- 2.4.2 Page Titled (A) - Descriptive page titles
- 2.4.3 Focus Order (A) - Logical focus order
- 2.4.4 Link Purpose (A) - Link purpose clear from text
- 2.4.5 Multiple Ways (AA) - Multiple ways to find pages
- 2.4.6 Headings and Labels (AA) - Descriptive headings/labels
- 2.4.7 Focus Visible (AA) - Keyboard focus indicator visible
- 2.4.11 Focus Not Obscured (AA) - Focus not completely hidden
- 2.4.13 Focus Appearance (AAA) - Not required for AA

**2.5 Input Modalities**
- 2.5.1 Pointer Gestures (A) - Path-based gestures have alternative
- 2.5.2 Pointer Cancellation (A) - Up-event or abort/undo
- 2.5.3 Label in Name (A) - Accessible name contains visible label
- 2.5.4 Motion Actuation (A) - Motion-triggered functions have UI
- 2.5.7 Dragging Movements (AA) - Drag operations have alternative
- 2.5.8 Target Size (AA) - 24x24px minimum (44x44px recommended)

### 3. Understandable

**3.1 Readable**
- 3.1.1 Language of Page (A) - Page language identified
- 3.1.2 Language of Parts (AA) - Part language identified

**3.2 Predictable**
- 3.2.1 On Focus (A) - No context change on focus
- 3.2.2 On Input (A) - No context change on input
- 3.2.3 Consistent Navigation (AA) - Navigation consistent
- 3.2.4 Consistent Identification (AA) - Components labeled consistently
- 3.2.6 Consistent Help (A) - Help in consistent location

**3.3 Input Assistance**
- 3.3.1 Error Identification (A) - Errors identified and described
- 3.3.2 Labels or Instructions (A) - Labels provided
- 3.3.3 Error Suggestion (AA) - Error correction suggested
- 3.3.4 Error Prevention (AA) - Reversible/checked/confirmed
- 3.3.7 Redundant Entry (A) - Auto-populated or selectable
- 3.3.8 Accessible Authentication (AA) - No cognitive function test

### 4. Robust

**4.1 Compatible**
- 4.1.2 Name, Role, Value (A) - For all UI components
- 4.1.3 Status Messages (AA) - Programmatically determined
